### Vue3基础面试题通常涵盖响应式原理、Composition API、生命周期钩子、模板语法、组件通信等核心概念
#### 一、响应式系统
- Vue3如何实现响应式？
基于Proxy对象拦截数据读写，替代Vue2的Object.defineProperty。
支持数组和对象的深层响应式，性能更优。
通过Reactive和Ref创建响应式数据。
- Reactive与Ref的区别？
Reactive：用于对象或数组，返回响应式代理对象。
Ref：用于基本类型或对象，需通过.value访问值，模板中自动解包。
#### 二、Composition API
- Setup函数的作用？
组件创建前执行，用于组合逻辑，替代Options API的data、methods等选项。
可返回响应式数据、函数、计算属性等。
- 什么是组合式函数（Composable）？
封装可复用逻辑的函数，如useFetch、useCounter，通过调用返回响应式状态和方法。
#### 三、生命周期钩子
- Vue3生命周期钩子有哪些？
onBeforeMount、onMounted、onBeforeUpdate、onUpdated、onBeforeUnmount、onUnmounted等。
替代Vue2的beforeMount、mounted等选项式生命周期。
- 为什么推荐在Setup中使用生命周期钩子？
更清晰的逻辑组织，避免选项式API中生命周期分散的问题。
#### 四、模板与指令
- Vue3模板语法有哪些改进？
支持多根节点（Fragment），不再强制单根元素。
指令如v-if、v-for、v-bind、v-on等用法与Vue2一致，但性能优化。
- v-model如何实现双向绑定？
本质是v-bind:value和v-on:input的语法糖，支持自定义组件的v-model。
#### 五、组件通信
- 父子组件如何通信？
父传子：Props传递数据，子组件通过defineProps接收。
子传父：通过emit触发事件，父组件监听事件。
provide/inject：跨层级传递数据。
- 什么是Teleport？
将组件内容渲染到DOM其他位置，如模态框、弹出层。
#### 六、其他基础考点
- Vue3的虚拟DOM如何优化？
静态节点提升、事件缓存、Fragment支持，提升渲染性能。
- 什么是Suspense？
处理异步依赖的组件，如异步组件加载时显示 fallback 内容。

#### 这 40 个问题的参考答案我都整理好了，内容比较详实，涵盖了从 ES6 基础到 Vite 构建工具的各个核心点。你可以把它们当作一份速查手册，配合着之前的题目进行自测和查漏补缺：

🟢 模块一：JavaScript 基础 (ES6+)

1.  var vs let/const
    -   作用域：var 是函数作用域，let/const 是块级作用域（{} 内有效）。
    -   变量提升：var 会变量提升（可以在声明前使用，值为 undefined），let/const 存在暂时性死区（声明前使用会报错）。
    -   重复声明：var 允许，let/const 不允许。
    -   修改：let 可修改，const 声明基本数据类型不可修改，声明对象时指针不可改，但属性可改。

2.  箭头函数 vs 普通函数
    -   this 指向：箭头函数没有自己的 this，它会捕获其所在上下文的 this 作为自己的 this（词法作用域）。普通函数的 this 指向调用它的对象。
    -   构造函数：箭头函数不能用作构造函数（不能用 new）。
    -   arguments：箭头函数没有 arguments 对象。

3.  解构赋值
    -   用法：const { name, age } = user; 或 const [a, b] = arr;。
    -   别名：const { name: userName } = user;（将 user.name 赋值给 userName）。
    -   默认值：const { name = 'Guest' } = user;（如果 name 不存在，则为 'Guest'）。

4.  模板字符串
    -   使用反引号   。
    -   可以直接换行。
    -   可以直接嵌入变量 ${variable}，无需拼接。

5.  Promise
    -   用于处理异步操作。
    -   三种状态：Pending（进行中）、Fulfilled（已成功）、Rejected（已失败）。
    -   状态一旦改变就不会再变（不可逆）。

6.  async / await
    -   async 函数返回一个 Promise。
    -   await 只能在 async 函数内部使用，它会暂停代码执行，等待 Promise 结果，使异步代码写起来像同步代码。

7.  模块化
    -   导出：export default（默认导出，一个文件只能有一个），export（按需导出，可有多个）。
    -   导入：import A from './A'（对应默认导出），import { B } from './B'（对应按需导出）。

8.  数组方法
    -   map：遍历数组，返回一个新数组（元素经过处理）。
    -   filter：遍历数组，返回满足条件的新数组。
    -   reduce：累积器，将数组所有元素汇总为一个值（如求和）。

9.  展开运算符 (...)
    -   合并：const newArr = [...arr1, ...arr2]。
    -   浅拷贝：const newObj = { ...oldObj }。
    -   传参：将数组展开为函数参数。

10. 闭包
    -   定义：函数嵌套函数，内部函数引用了外部函数的变量。
    -   作用：数据私有化、保存状态。
    -   缺点：滥用会导致内存泄漏（变量无法被回收）。

🔵 模块二：Vue 3 核心概念

11. Vue 2 vs Vue 3
    -   API 风格：Vue 2 是选项式 API (Options API)，Vue 3 新增组合式 API (Composition API)。
    -   响应式原理：Vue 2 用 Object.defineProperty，Vue 3 用 Proxy。
    -   性能：Vue 3 打包体积更小，渲染更快。

12. MVVM
    -   Model：数据模型。
    -   View：视图（UI）。
    -   ViewModel：连接 View 和 Model 的桥梁（Vue 实例），通过双向数据绑定和事件监听自动同步。

13. 响应式原理 (Vue 3)
    -   利用 ES6 的 Proxy 对象拦截对数据的读取和设置操作。
    -   配合 Reflect 操作源对象。
    -   相比 Vue 2，可以监听数组变化、对象属性新增/删除，性能更好。

14. ref vs reactive
    -   ref：主要用于定义基本数据类型（数字、字符串），也可以定义对象。在 JS 中访问需要 .value，模板中自动解包。
    -   reactive：主要用于定义对象或数组。它是深层响应式的，访问不需要 .value。

    -   其中定义的顶层变量、函数会自动暴露给模板使用，无需 return。

16. 虚拟 DOM
    -   用 JavaScript 对象模拟真实 DOM 树。
    -   作用：在内存中进行 Diff 算法对比，计算出最小变更量，再批量更新真实 DOM，减少重绘重排，提高性能。

17. 生命周期
    -   onMounted：DOM 挂载完成后执行（常用于发请求、操作 DOM）。
    -   onUpdated：数据更新导致 DOM 更新后。
    -   onUnmounted：组件销毁前（清理定时器、事件监听）。

18. v-if vs v-show
    -   v-if：真正的条件渲染。条件为假时元素不在 DOM 中。切换开销大（涉及销毁/重建）。
    -   v-show：元素始终在 DOM 中，只是切换 CSS 的 display 属性。初始渲染开销大，切换开销小。

19. key 的作用
    -   在 v-for 中用于唯一标识节点。
    -   帮助 Vue 的 Diff 算法高效地判断哪些元素被修改、添加或移除，避免复用错误的 DOM 状态（如输入框内容错乱）。

20. computed vs methods
    -   computed：有缓存。只有依赖的数据变化时才会重新计算，否则直接返回缓存值。
    -   methods：无缓存。每次触发或渲染都会重新执行函数。

🟠 模块三：组件化开发

21. 组件化好处
    -   复用性、可维护性、代码解耦。

22. 父传子 (props)
    -   父组件在标签上绑定属性，子组件用 defineProps 接收。

23. 子传父 (emits)
    -   子组件用 defineEmits 定义事件，通过 emit('eventName', data) 触发。父组件监听该事件。

24. 单向数据流
    -   数据只能从父流向子。
    -   原因：防止子组件意外修改父组件状态，导致数据流向混乱，难以调试。

25. 修改 Props 对象
    -   如果是对象，修改内部属性是可以的（因为是引用传递），但不能直接替换整个对象。
    -   最佳实践：使用 v-model 或触发事件让父组件修改。

26. 插槽

    -   具名插槽：，用于指定位置插入内容。
    -   作用域插槽：子组件将数据传给插槽，父组件在插槽中接收并使用。

27. 生命周期顺序
    -   父 created -> 子 created -> 子 mounted -> 父 mounted。

28. 动态组件

    -   常用于标签页切换。配合  可以缓存状态。

29. provide / inject
    -   祖先组件 provide 数据，后代组件 inject 数据。
    -   解决深层嵌套组件传值（透传）问题。

30. 自定义指令

🟣 模块四：路由与状态管理

31. Hash vs History
    -   hash：URL 带 #，兼容性好，不需要后端配置。
    -   history：URL 好看（标准路径），刷新页面需要后端配合（否则 404）。

32. 动态路由
    -   配置 path: '/user/:id'。
    -   获取：route.params.id。

33.  vs 
    -   link：生成跳转链接（渲染为  标签）。
    -   view：显示当前路由匹配到的组件内容（占位符）。

34. 编程式导航
    -   router.push('/home')：跳转。
    -   router.back()：后退。

35. 路由守卫
    -   router.beforeEach((to, from, next) => { ... })。
    -   常用于判断 token，没登录则重定向到登录页。

36. 为什么用 Pinia
    -   当多个组件共享状态，或者组件层级太深时，用 Props 传值太麻烦。
    -   全局单例状态管理。

37. Pinia 核心
    -   state：数据。
    -   getters：计算属性（派生状态）。
    -   actions：方法（支持异步）。

38. Pinia vs Vuex
    -   Pinia 去掉了 mutations`（直接改 state）。
    -   对 TypeScript 支持更好。
    -   体积更小。
